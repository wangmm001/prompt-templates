## 📖 什么是提示词工程

提示词工程 (Prompt Engineering) 是设计和优化输入给 AI 的提示词，以获得高质量输出的艺术和科学。

### 为什么重要

- 🎯 **提高输出质量**: 好的提示词产生更好的结果
- ⚡ **减少迭代次数**: 一次到位，节省时间
- 💰 **降低成本**: 减少 token 消耗
- 🔒 **增强可控性**: 更精确地控制 AI 行为

## 🏗️ 核心框架

### CO-STAR 框架

```
C - Context (上下文): 提供背景信息
O - Objective (目标): 明确任务目标
S - Style (风格): 指定写作风格
T - Tone (语气): 设定语气基调
A - Audience (受众): 定义目标读者
R - Response (响应): 指定输出格式
```

#### 示例

```markdown
# Context
你是一名资深网络安全分析师，正在为 CISO 准备季度安全报告。

# Objective
分析本季度网络流量中的异常模式，识别潜在威胁。

# Style
专业、简洁、数据驱动

# Tone
严肃但不危言耸听

# Audience
首席信息安全官 (CISO) 和技术总监

# Response
- 执行摘要 (200 字)
- 关键发现 (3-5 点)
- 风险评级 (高/中/低)
- 建议措施
```

---

### CREATE 框架

```
C - Character (角色): 定义 AI 扮演的角色
R - Request (请求): 清晰的任务请求
E - Examples (示例): 提供示例参考
A - Adjustments (调整): 指定调整和优化
T - Type (类型): 输出类型说明
E - Extras (额外): 其他补充信息
```

#### 示例

```markdown
# Character
你是一位有 10 年经验的流量分析专家

# Request
请分析以下流量数据，识别异常模式

# Examples
正常模式：工作日 9-18 点流量高峰，夜间低谷
异常模式：深夜大量数据外传，周末异常连接

# Adjustments
重点关注加密流量和非常规端口

# Type
结构化报告，包含表格和图表说明

# Extras
数据时间范围：2025-Q4
网络规模：5000+ 设备
```

## 📝 提示词结构

### 标准结构

```markdown
# Role: [角色定义]

- 经验：[经验水平]
- 专长：[核心技能]

## Context
[背景信息和场景描述]

## Task
[具体任务描述]
1. 子任务 1
2. 子任务 2
3. 子任务 3

## Output Format
[期望的输出格式]
- 结构要求
- 长度要求
- 格式要求

## Constraints
[约束条件]
- 必须遵守的规则
- 避免的事项
- 质量标准

## Examples
[示例 - 可选但推荐]
好的示例和不好的示例对比
```

## 🎯 最佳实践

### ✅ DO - 应该做的

1. **明确角色定义**
   ```markdown
   ✅ 你是一位有 10 年经验的网络安全专家
   ❌ 你是专家
   ```

2. **提供充分上下文**
   ```markdown
   ✅ 企业网络规模 5000 人，日均流量 10TB，主要挑战是加密流量检测
   ❌ 分析网络流量
   ```

3. **指定输出格式**
   ```markdown
   ✅ 输出包含：执行摘要、3 个关键发现、风险评级、5 条建议
   ❌ 给出分析结果
   ```

4. **设定量化标准**
   ```markdown
   ✅ 准确率>95%，延迟<100ms，输出 3000-5000 字
   ❌ 效果好一点，字数适中
   ```

5. **提供示例**
   ```markdown
   ✅ 好的输出示例：[具体示例]
      不好的输出示例：[具体示例]
   ❌ 无示例
   ```

6. **使用结构化格式**
   ```markdown
   ✅ 使用标题、列表、表格
   ❌ 大段文字描述
   ```

---

### ❌ DON'T - 应该避免的

1. **模糊指令**
   ```markdown
   ❌ 分析一下这个
   ✅ 分析以下流量数据中的异常模式
   ```

2. **多重否定**
   ```markdown
   ❌ 不要不考虑到可能的例外情况
   ✅ 请考虑所有例外情况
   ```

3. **过于宽泛**
   ```markdown
   ❌ 告诉我关于网络安全的一切
   ✅ 列出 2025 年最重要的 5 个网络安全趋势
   ```

4. **缺少约束**
   ```markdown
   ❌ 写一份报告
   ✅ 写一份 2000 字以内的技术报告，包含执行摘要和 3 个主要发现
   ```

5. **假设 AI 知道上下文**
   ```markdown
   ❌ 继续之前的分析
   ✅ 基于以下数据继续分析：[数据]
   ```

## 🔧 高级技巧

### 1. 思维链 (Chain of Thought)

让 AI 展示推理过程：

```markdown
请逐步分析：
1. 首先，识别数据中的异常模式
2. 然后，评估每个异常的严重性
3. 接着，分析可能的原因
4. 最后，给出缓解建议

请展示你的推理过程。
```

### 2. 少样本学习 (Few-Shot Learning)

提供少量示例帮助 AI 理解任务：

```markdown
示例 1:
输入：[正常流量模式描述]
输出：正常 - 符合工作日模式

示例 2:
输入：[异常流量模式描述]
输出：异常 - 深夜大量外传

现在分析：[新数据]
```

### 3. 角色嵌套

使用多层角色：

```markdown
你是一位资深安全分析师。请以以下三个视角分析问题：
1. 技术视角：检测技术和工具
2. 业务视角：对业务的影响
3. 管理视角：资源和人员需求
```

### 4. 自我验证

让 AI 自我检查：

```markdown
完成分析后，请：
1. 检查是否所有要求都满足
2. 验证数据是否准确
3. 确认建议是否可执行
4. 指出分析的局限性
```

### 5. 迭代优化

分阶段获取输出：

```markdown
第一阶段：输出大纲
第二阶段：基于反馈完善
第三阶段：最终版本
```

## 📈 质量评估

### 评估维度

| 维度 | 权重 | 检查项 |
|-----|------|--------|
| 清晰度 | 30% | 无歧义、逻辑清晰 |
| 完整性 | 25% | 要素齐全、信息充分 |
| 可执行性 | 25% | AI 能准确理解并执行 |
| 输出质量 | 20% | 结果符合预期 |

### 评分标准

- ⭐⭐⭐⭐⭐ (90-100): 优秀，可直接使用
- ⭐⭐⭐⭐ (75-89): 良好，少量优化
- ⭐⭐⭐ (60-74): 可用，需要调整
- ⭐⭐ (40-59): 需改进
- ⭐ (<40): 重新设计

## 🔍 常见问题

### Q1: 提示词多长合适？

**A**: 没有固定标准，但建议：
- 简单任务：100-300 字
- 复杂任务：500-1500 字
- 超长提示词可能影响效果

### Q2: 如何平衡详细度和简洁性？

**A**: 
- 核心指令简洁
- 详细说明放注释
- 使用分层结构

### Q3: 为什么 AI 不遵循指令？

**A**: 可能原因：
- 指令模糊或有歧义
- 缺少约束条件
- 任务超出 AI 能力
- 需要更多上下文

### Q4: 如何提高输出一致性？

**A**:
- 明确输出格式
- 提供示例
- 使用温度参数 (temperature=0)
- 添加验证步骤
