# 提示词模板质量评价框架

**版本**: v1.0  
**更新日期**: 2026-03-15

---

## 评价维度

### 1. 结构完整性 (25 分)

评估模板是否包含提示词的核心要素。

| 分值 | 标准 |
|-----|------|
| 23-25 | 包含所有核心要素：角色、上下文、任务、输出格式、约束条件 |
| 18-22 | 包含 4 个核心要素，缺少 1 个 |
| 13-17 | 包含 3 个核心要素 |
| 8-12  | 包含 2 个核心要素 |
| 0-7   | 缺少超过 3 个核心要素 |

**核心要素清单**:
- [ ] 角色定义 (Role/Profile)
- [ ] 上下文/背景 (Context/Background)
- [ ] 任务描述 (Task)
- [ ] 输出格式 (Output Format)
- [ ] 约束条件 (Constraints)
- [ ] 示例 (Examples) - 加分项

---

### 2. 可执行性 (25 分)

评估模板指令是否清晰、可操作、无歧义。

| 分值 | 标准 |
|-----|------|
| 23-25 | 指令极其清晰，任何 AI 都能准确理解并执行 |
| 18-22 | 指令清晰，少量模糊但不影响执行 |
| 13-17 | 大部分指令清晰，部分需要推断 |
| 8-12  | 指令模糊，需要大量推断 |
| 0-7   | 指令混乱，难以执行 |

**检查要点**:
- [ ] 动词使用准确（"分析"vs"看看"）
- [ ] 数量词明确（"3 个案例"vs"一些案例"）
- [ ] 无歧义表达
- [ ] 逻辑顺序清晰
- [ ] 任务可分解

---

### 3. 领域专业性 (20 分)

评估模板是否体现领域专业知识和术语。

| 分值 | 标准 |
|-----|------|
| 18-20 | 专业术语准确，体现深度领域知识 |
| 15-17 | 术语使用正确，有一定专业性 |
| 11-14 | 基本术语正确，专业性一般 |
| 6-10  | 术语使用有误或不一致 |
| 0-5   | 缺乏专业性 |

**检查要点**:
- [ ] 术语使用准确
- [ ] 概念定义清晰
- [ ] 符合行业最佳实践
- [ ] 参考权威来源
- [ ] 避免过度简化

---

### 4. 灵活性 (15 分)

评估模板是否支持参数化、可扩展、适配不同场景。

| 分值 | 标准 |
|-----|------|
| 14-15 | 高度参数化，易于适配多种场景 |
| 11-13 | 支持参数化，适配性良好 |
| 8-10  | 部分参数化，需要修改才能适配 |
| 5-7   | 参数化程度低 |
| 0-4   | 固定内容，无法适配 |

**检查要点**:
- [ ] 使用 `{{parameter}}` 占位符
- [ ] 参数说明完整
- [ ] 支持场景扩展
- [ ] 模块化设计
- [ ] 可组合使用

---

### 5. 输出质量 (15 分)

评估预期输出是否结构化、可验证、有价值。

| 分值 | 标准 |
|-----|------|
| 14-15 | 输出格式清晰，结果高度可用 |
| 11-13 | 输出格式明确，结果基本可用 |
| 8-10  | 有输出格式要求，但不够具体 |
| 5-7   | 输出格式模糊 |
| 0-4   | 无输出格式要求 |

**检查要点**:
- [ ] 输出结构明确
- [ ] 包含表格/列表等结构化元素
- [ ] 有字数或长度要求
- [ ] 有质量检查标准
- [ ] 提供示例输出

## 综合评价

### 评分等级

| 等级 | 分数范围 | 说明 | 建议 |
|-----|---------|------|------|
| ⭐⭐⭐⭐⭐ 优秀 | 90-100 | 可直接用于生产环境 | 推荐使用 |
| ⭐⭐⭐⭐ 良好 | 75-89 | 少量修改即可使用 | 推荐使用 |
| ⭐⭐⭐ 可用 | 60-74 | 需要适当调整 | 修改后使用 |
| ⭐⭐ 需改进 | 40-59 | 需要大量修改 | 谨慎使用 |
| ⭐ 不推荐 | <40 | 建议重新设计 | 不推荐使用 |

### 评分计算

```
总分 = 结构完整性 (25) + 可执行性 (25) + 领域专业性 (20) + 灵活性 (15) + 输出质量 (15)
```

## 评价流程

### 步骤 1: 初步审查
- 快速浏览模板整体结构
- 检查是否包含核心要素
- 标记明显问题

### 步骤 2: 详细评估
- 逐维度打分
- 记录扣分原因
- 提出改进建议

### 步骤 3: 实测验证
- 使用模板实际测试
- 检查 AI 输出质量
- 验证是否达到预期

### 步骤 4: 综合评定
- 计算总分
- 确定评级
- 编写评价报告

## 常见问题

### Q1: 如何平衡模板的详细程度？

**A**: 模板应足够详细以确保输出质量，但不应过于冗长影响可读性。建议：
- 核心指令简洁明了
- 详细说明放在注释或附录
- 使用分层结构（概要 + 详细）

### Q2: 参数化程度如何把握？

**A**: 参数化应服务于灵活性，但过度参数化会增加使用复杂度。建议：
- 核心变量必须参数化
- 提供合理的默认值
- 参数数量控制在 10 个以内

### Q3: 如何处理领域差异？

**A**: 不同领域对专业性要求不同。建议：
- 通用模板保持适度专业
- 专业模板深入领域细节
- 提供领域术语表
