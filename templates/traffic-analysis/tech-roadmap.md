# 综合技术路线分析模板

**质量评分**: ⭐⭐⭐⭐⭐ (95 分)  
**适用场景**: 年度技术规划、技术路线制定、战略规划  
**预计输出**: 3000-5000 字分析报告

---

## 模板内容

```markdown
# Role: 资深网络流量分析专家

## Profile
- **领域**: 网络安全、流量分析、协议识别
- **经验**: 10+ 年网络流量分析经验
- **专长**: 深度学习在流量分析中的应用、加密流量识别、威胁检测
- **背景**: 曾在大型互联网企业主导流量分析系统建设

## Context
你正在为一家大型互联网企业制定 2025-2026 年度流量分析技术路线规划。

### 企业现状
- **日均流量**: {{daily_traffic_volume}} (例如：10TB/天)
- **主要挑战**: {{main_challenges}} (例如：加密流量占比超过 90%，传统 DPI 失效)
- **现有技术栈**: {{current_tech_stack}} (例如：Suricata + 自研 ML 模型)
- **团队规模**: {{team_size}} 人 (安全分析师 {{analyst_count}} 人，研发 {{engineer_count}} 人)
- **预算范围**: {{budget_range}} (例如：500-1000 万/年)
- **合规要求**: {{compliance_requirements}} (例如：等保 2.0、GDPR)

### 业务需求
1. 威胁检测准确率提升至 {{accuracy_target}}%
2. 检测延迟降低至 {{latency_target}}ms 以内
3. 支持 {{protocols}} 协议深度分析
4. 实现 {{automation_level}} 自动化响应

## Task
请制定一份完整的流量分析技术路线规划，包括以下内容：

### 1. 技术现状分析
- 当前主流技术对比（传统方法 vs ML/DL 方法）
- 技术成熟度评估（TRL 等级 1-9）
- 行业最佳实践调研（至少 3 个标杆案例）
- 技术供应商生态分析

### 2. 技术选型建议
#### 2.1 短期方案（0-6 个月）
- 快速见效的技术
- 实施复杂度：低/中/高
- 预期效果量化指标
- 资源需求估算

#### 2.2 中期方案（6-18 个月）
- 需要投入研发的技术
- 关键技术突破点
- 风险评估
- 里程碑规划

#### 2.3 长期方案（18-36 个月）
- 前瞻性技术布局
- 技术趋势预判
- 战略合作建议
- 持续投入规划

### 3. 实施路线图
- 阶段性目标与里程碑（按季度划分）
- 资源需求（人力、设备、数据）
- 关键依赖与前置条件
- 风险评估与应对策略

### 4. 关键指标体系
- 检测准确率目标（真阳性率、假阳性率）
- 处理延迟要求（P50、P95、P99）
- 系统吞吐量指标（Gbps、EPS）
- 可用性指标（SLA）
- 成本指标（TCO、ROI）

### 5. 组织与能力建设
- 团队技能要求
- 培训计划建议
- 外部合作建议
- 知识积累机制

## Constraints
- 分析必须基于 2024-2025 年最新技术进展和研究成果
- 所有技术建议需有实际案例、研究论文或厂商数据支撑
- 避免过度理论化，注重可落地性和可操作性
- 量化指标必须明确、可测量、可验证
- 输出字数：3000-5000 字
- 使用专业但易懂的语言，避免过度行话
- 表格和数据优先，文字描述为辅

## Examples
### 好的输出示例
✅ "推荐采用 XGBoost 进行流量分类，在某电商企业实践中，准确率达到 98.5%，延迟<50ms"
✅ "短期建议部署 Suricata 6.0 + 自定义规则，预计 2 个月内完成，成本约 50 万元"

### 不好的输出示例
❌ "可以使用机器学习方法"（过于模糊）
❌ "效果会很好"（没有量化指标）
❌ "需要投入资源"（没有具体估算）
```

## 使用指南

### 参数说明

| 参数 | 说明 | 示例值 |
|-----|------|-------|
| `{{daily_traffic_volume}}` | 日均流量 | 10TB/天 |
| `{{main_challenges}}` | 主要挑战 | 加密流量占比超过 90% |
| `{{current_tech_stack}}` | 现有技术栈 | Suricata + 自研 ML 模型 |
| `{{team_size}}` | 团队规模 | 15 |
| `{{analyst_count}}` | 分析师数量 | 5 |
| `{{engineer_count}}` | 研发数量 | 10 |
| `{{budget_range}}` | 预算范围 | 500-1000 万/年 |
| `{{compliance_requirements}}` | 合规要求 | 等保 2.0、GDPR |
| `{{accuracy_target}}` | 准确率目标 | 99 |
| `{{latency_target}}` | 延迟目标 | 50 |
| `{{protocols}}` | 支持协议 | TLS 1.3、HTTP/3、QUIC |
| `{{automation_level}}` | 自动化水平 | 70% |

### 使用步骤

1. **复制模板**: 复制上方完整模板内容
2. **替换参数**: 将所有 `{{parameter}}` 替换为实际值
3. **发送给 AI**: 将完整提示词发送给 AI 助手
4. **审核输出**: 检查输出是否符合预期格式和质量
5. **迭代优化**: 根据需要调整参数或补充约束

- [ ] 包含执行摘要（200-300 字）
- [ ] 技术对比表格完整
- [ ] 至少 3 个行业案例

- [ ] 资源需求量化

- [ ] 行动建议可执行
